01

Creating your account

The very first screen anyone sees, before they belong to any organization.

Go to the site link your admin sent you. You'll land on the sign-in screen. If this is your first time, switch to Create Account instead — enter any email address you actually have access to, and a password of at least 6 characters.

02

Your organization

Every shipment, booking, and teammate lives inside an organization. You'll see this screen right after your first sign-in.

You have two options here, and you'll only need one of them:

  • Starting a new company workspace? Use Create Organization — you become its first member automatically.
  • Joining a teammate's existing workspace? Ask them for their invite code (they'll find it in their sidebar — see Section 3) and use Join with Invite Code instead.
If you already belong to one or more organizations, this screen instead shows a grid of them to pick from — see Section 6, "Multiple organizations."
03

The dashboard

Home base. Everything else in the app is reachable from here.

04

Creating a booking

One form, three shapes — the fields change based on how the cargo is moving.

Click + New Booking from the dashboard. Shipper, Consignee, Origin, and Destination are required for every booking regardless of mode — the Create Booking button stays disabled until all four are filled in. Everything below that changes depending on which tab you're on.

Ocean — FCL or LCL

Air — with automatic volumetric weight

volumetric weight (kg) = (length × width × height in cm) ÷ 6000
chargeable weight (kg) = max(gross weight, volumetric weight)

Truck — domestic road moves

A Truck booking starts life with status Loading instead of Booked — everything else starts as Booked. This is the app's only automatic status assignment right now; moving a shipment to Cleared, In Transit, etc. isn't wired up to a button yet (see Section 8).
05

Finding a shipment

No dedicated search page — it all happens right on the dashboard.

Type into the search bar at the top of the dashboard — it matches against the tracking reference, client name, origin, and destination simultaneously, updating the table as you type. Combine it with the mode filter tabs (All / Ocean / Air / Truck) to narrow further — for example, typing "Rotterdam" while the Ocean tab is selected shows only ocean shipments bound for Rotterdam.

06

Working across multiple organizations

If you're on staff at more than one company using SST Freight, or manage several branches as separate workspaces.

Click ⇄ Switch organization in the sidebar at any time. You'll see a grid of every organization you belong to, each showing your role in it — pick one to jump straight to its dashboard. Your data never mixes between organizations: shipments, bookings, and search results you see are always scoped to whichever one is currently selected.

The app remembers your last choice, so next time you sign in you'll land back in the same organization automatically.

07

Who can do what, today

An honest answer, not an aspirational one.

Every organization has an owner (whoever created it) and members (everyone who joined via invite code). Right now, in this first release, that distinction is visible but not yet enforced — every member, regardless of role, can view and create bookings within the organization. There's no separate "read-only" or "admin-only" mode yet.

What every member can do

  • View all shipments in the organization
  • Create new bookings (Ocean, Air, Truck)
  • Search and filter the shipment list
  • See and share the organization's invite code
  • Switch between organizations they belong to

What's role-specific today

  • Nothing yet — only the badge next to your name (owner / member) differs
  • Anyone can invite new teammates by sharing the code
  • There's no way to remove a member or transfer ownership yet
Only share your invite code with people you actually want inside your organization — anyone with the code can join and see every shipment in it.
08

What's coming later

Two sections are visible in navigation today as a preview of the roadmap.

  • Directory — a proper record of shippers, consignees, and overseas agents, instead of typing names freehand into each booking.
  • Customs Filings — Bill of Entry and Shipping Bill filing workflows.

Clicking either from the sidebar today shows a short placeholder message rather than an error — that's expected, not a bug.

09

Troubleshooting

The handful of things people actually get stuck on.

I signed up and got "Email not confirmed" when trying to sign in

Ask your admin to confirm your account from the Supabase dashboard, or wait for a confirmation email if one was sent. Once your admin has this fully configured, new signups sign you in immediately with no extra step.

I can't see a shipment my colleague just created

Make sure you're both looking at the same organization — check the name at the top of the sidebar on both screens. If they match and it's still missing, try the search box in case a filter is narrowing the list, then a full page refresh.

The dashboard shows "Couldn't load shipments" with a Retry button

This means the app successfully reached the server but the request failed — usually a brief connectivity issue. Click Retry. If it keeps happening, tell your admin.

I created the wrong organization by mistake

There's no delete-organization option in this release. Create the correct one and simply stop using the old one — or ask your admin to clean it up directly in the database.

Can I change my password?

Not from within the app yet in this release — talk to your admin.
